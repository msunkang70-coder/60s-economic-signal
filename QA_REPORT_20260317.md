### CRITICAL-01: TypeError 크래시 — 다운로드 섹션 전체 사망
- **위치**: `app.py:1162` `generate_report_html()`
- **현상**: 소비재 after p8 하단에 빨간 에러 박스 "TypeError: can only concatenate str (not 'dict') to str"
- **원인**: `summary_3lines`가 4-frame dict로 변경되었으나 `generate_report_html()`에서 여전히 str 연결 시도
- **영향**: 리포트 다운로드 불가, 페이지 하단 전체 렌더링 중단
- **코드**:
  ```python
  # L1162 (현재 — 크래시)
  full = sel_doc["title"] + " " + detail.get("summary_3lines", "")
  # summary_3lines가 dict {"impact": "...", "risk": "...", ...}이면 TypeError
  ```
- **수정안**: dict일 때 values를 join하여 str로 변환
- **우선순위**: P0 (즉시)
- **난이도**: ★☆☆ (5분)

### MAJOR-02: 4-frame 요약 마크다운 raw 노출
- **위치**: `app.py:4218-4228`
- **현상**: 소비재 after p5에서 "**K-뷰티**", "**글로벌 유통**", "**브랜드 가치**" 등이 `**텍스트**` 형태로 그대로 표시
- **원인**: `st.html()`은 HTML만 렌더링하고 Markdown 문법은 처리하지 않음. LLM(Groq)이 응답에 `**볼드**` Markdown을 사용하나, 이를 HTML `<b>` 태그로 변환하지 않고 raw 삽입
- **영향**: 전문성·가독성 저하
- **수정안**: 렌더링 전 `**text**` → `<b>text</b>` 변환
  ```python
  import re
  _ft = re.sub(r'\*\*(.+?)\*\*', r'<b>\1</b>', _ft)
  ```
- **우선순위**: P1
- **난이도**: ★☆☆ (5분)

## 3. 터미널 로그 기반 추가 발견

### LOG-01: KOTRA URL 복원율 극저 (0~6.7%)
```
P5 복원율: 0/8 (0.0%) [cache=0 gnews=0 title=0 fail=8]   ← 일반
P5 복원율: 1/15 (6.7%) [cache=1 gnews=0 title=1 fail=14]  ← 소비재
```
- KOTRA Google News RSS에서 실제 KOTRA URL을 복원하는 비율이 극히 낮음
- 복원 실패 시 `gnews_url` (no_fetch) 모드로 전환 → 본문 추출 불가
- **우선순위**: P2
- **난이도**: ★★★ (KOTRA API 또는 HTML 파싱 구조 개선 필요)

### LOG-02: Groq Rate Limit 빈번
```
🚦 Rate Limit (429) [Llama3.3-70B] — 즉시 다음 모델로 패스
🔄 기본 모델 실패 → 경량 모델 [Llama3.1-8B] 시도
❌ [Llama3.1-8B] API 오류 413: Request too large (TPM Limit 6000, Requested 6075)
```
- 70B 모델 429 → 8B 모델 413 → smart_fallback으로 품질 75점 요약 사용
- **우선순위**: P2
- **난이도**: ★★☆ (본문 길이 트리밍 또는 토큰 예산 관리)

### LOG-03: 산업부 RSS ConnectionReset 불안정
```
motie_source] RSS 실패: ConnectionResetError(10054, '원격 호스트에 의해 강제로 끊김')
motie_source] HTML 파서 fallback 시도
```
- korea.kr RSS가 간헐적으로 연결 거부 → HTML fallback은 동작하나 안정성 문제
- **우선순위**: P3
- **난이도**: ★☆☆ (이미 fallback 구현됨, 모니터링만 추가)
